## Git을 배워야 하는 이유

Git을 왜 배워야 할까? 우리가 코딩을 한다면 Git은 마치 필수 전공과목과 같다. 개발자라면 어느 분야에서든지 현업에서 일하기 위해 당연히 Git을 쓸 줄 알아야 하고 AI를 활용해서 바이브 코딩으로 프로그램을
만드시는 분들도 git을 알아두는 것이 좋다. 사용법을 다 익히지는 않아도 git에는 어떤 기능이 있고 각각이 무엇을 위함인지 알아두면 AI를 통해 git을 쉽게 사용할 수 있을 것이다.

git과 같은 프로그램을 **VCS** 라고 부른다. 바로 "버전 관리 시스템"의 약자이다. 쉽게 말해서 우리가 개발하고 있는 프로그램을 중간중간마다 압축 파일같은 것으로 백업해주는 프로그램이라고 생각하면 좋을
것이다. 이런 백업본들 각각이 프로그램의 **버전** 이 되는 것이다. 그렇다고 실제로 큰 용량의 압축파일을 만드는 것은 아니고 개발 중 언제든지 과거시점으로 돌아갈 수 있도록 중요한 시점마다 세이브 포인트를 만드는
것이다. 마치 게임을 하다가 중간에 저장하듯이 말이다.

소프트웨어를 개발하다 보면 새로운 버전의 문제가 있거나 하는 이유로 프로그램을 과거의 특정 시점으로 되돌려야 하는 일이 자주 발생한다. 예를 들어 웹 어플리케이션에 특정 기능을 추가하였는데 알 수 없는 이유로 오류가
발생하여 그 원인을 파악하고자 일단 웹 어플리케이션이 정상 동작하던 마지막 시점으로 되돌려야 하는 경우가 될 것이다. 하지만 프로그래밍 작업은 여러 파일들과 복잡한 코드들을 통해 이뤄지기 때문에 소프트웨어의 상태를
직접 과거로 돌리는 것은 머리가 아플 것이다. 특히 AI를 활용하여 프로그램을 만드는 분들은 긴 과정을 거쳐서 만든 제품에 문제가 생겼을 때 제품이 규모가 크고 복잡해져서 AI가 이를 제대로 복구하지 못한다면 크게
난처해질 수 있다. 만약 깃을 사용해서 중요 포인트마다 버전을 세이브 해두었다면 언제든 그 시점으로 돌아갈 수 있다. 이 세이브 포인트들을 통해 언제든 시간여행을 할 수 있으니 마음 놓고 새로운 기능들을 개발할 수
있는 것이다.

깃은 이런 시간여행뿐만 아니라 여러 멀티버스를 뛰어 넘어다니는 차원여행도 가능하게 해준다. 프로그램을 개발하면서 다양한 버전으로 테스트 해보고 싶은 부분이 있거나 제품을 사용자의 종류 또는 용도마다 다른 형태나
기능으로 만들 필요가 있는 경우 프로그램을 원하는 시점에서 여러 갈래로 분기하여 다양한 버전을 개발할 수 있는 것이다.

깃을 사용하면 프로그램 폴더를 따로 따로 복사하지 않고도 이를 간편하게 할 수 있다.
