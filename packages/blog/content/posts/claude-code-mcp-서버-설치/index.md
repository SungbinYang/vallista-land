## 유용한 MCP Servers 설치

[MCP가 무엇인지](/claude-code-mcp-활용) 알았다면, 이제 *실무에서 정말 자주 쓰는* 서버들을 **[스타터킷](/claude-code-starter-kit-만들기-공식문서)에 미리 심어두는**
단계다.

> 자주 사용하는 MCP 서버를 *미리 설정해 놓는다면* 그만큼 **시간이 단축** 될 거예요.

매번 새 프로젝트마다 *똑같은 세팅* 을 반복할 필요가 없다. **스타터킷** 은 *개발을 빠르게 시작* 하도록 미리 구성한 템플릿이니, *공통으로 쓰는* MCP 서버는 **여기에 박아두는** 게 정석이다. 이번에
심을 서버는 셋이다.

### ① Context7 — 최신 문서를 끌어오는 서버

**Context7** 은 *라이브러리·프레임워크의 문서* 를 자동으로 분석해 **필요한 정보를 빠르게** 찾아준다.

LLM은 *학습 시점 이후* 의 API 변경을 모른다. 그래서 *예전 문법* 으로 코드를 짜는 일이 잦은데, Context7이 **최신 공식 문서** 를 그 자리에서 끌어와 *환각을
줄여준다.* [MCP 활용 챕터](/claude-code-mcp-활용)에서 이미 만난 그 서버다.

### ② Sequential Thinking — 단계별로 생각하게 만드는 서버

**Sequential Thinking** 은 *복잡한 문제* 를 **단계별로 분석·추론** 하도록 돕는다.

> 시퀀셜 띵킹 MCP 서버는 *복잡한 문제 해결 과정* 을 **단계별로 분석하고 추론** 하는 데 도움을 주는 MCP 서버예요.

*복잡한 로직 구현* 이나 *디버깅* 처럼 **생각의 흐름** 이 중요한 작업에서 빛난다. 클로드가 *한 번에 답으로 점프* 하지 않고, **차근차근** 단계를 밟게 만드는 것이다.

쓰는 방법은 둘이다.

- **자동** — *단계적 사고* 가 필요한 요청이면 클로드가 알아서 호출
- **명시적** — 프롬프트에 **`use sequential thinking`** 을 붙여 *강제로* 사용

강의에선 *"Next.js 전문가로서 코드베이스를 분석해 [`CLAUDE.md`](/claude-code-설정-파일과-메모리-관리)를 최적화해줘"* 같은, **여러 단계가 얽힌** 요청에 활용한다.

### ③ shadcn/ui — UI 컴포넌트를 정확히 가져오는 서버

**shadcn/ui MCP** 는 AI가 *shadcn/ui 컴포넌트* 를 **더 정확하게** 쓰도록 돕는 **공식 MCP 서버** 다.

*컴포넌트 문서* 와 *예제 코드* 를 그 자리에서 가져오니, *추측으로 짠 UI* 가 아니라 **공식 권장 스타일** 을 따른 결과가 나온다. 강의에선 이 서버로 *"깔끔하고 현대적인 반응형 로그인 페이지"* 를
만든다 — 필요한 컴포넌트를 **검색 → 공식 코드 스타일 준수 → 완성** 의 흐름이다.

### 어떻게 설치하나 — `.mcp.json`

세 서버 모두 **`.mcp.json`** 파일에 *JSON 설정* 을 더하는 방식으로 붙는다. ([프로젝트 스코프](/claude-code-mcp-활용)로 두면 *팀 전체* 가 같은 서버를 공유한다.)

```json
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": [
        "-y",
        "@upstash/context7-mcp"
      ]
    },
    "sequential-thinking": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-sequential-thinking"
      ]
    },
    "shadcn": {
      "command": "npx",
      "args": [
        "-y",
        "shadcn@latest",
        "mcp"
      ]
    }
  }
}
```

이렇게 **스타터킷에 세 서버를 박아두면**, 새 프로젝트를 열 때마다 *세팅을 다시 할 필요가 없다.*

### 정리하며

유용한 MCP 서버 설치를 정리하면 다음과 같다.

- **왜?** → 공통 MCP 서버를 [스타터킷](/claude-code-starter-kit-만들기-공식문서)에 미리 심어 → *매번 반복되는 설정 시간* 단축
- **Context7** → *최신 공식 문서* 를 끌어와 **환각 감소**
- **Sequential Thinking** → *복잡한 문제* 를 **단계별 추론** (`use sequential thinking` 으로 명시 호출)
- **shadcn/ui** → *공식 컴포넌트 문서·예제* 로 **정확한 UI** 생성 (공식 MCP)
- **설치** → [`.mcp.json`](/claude-code-mcp-활용)에 JSON 설정 추가

> 우리는 *매번 프로젝트를 진행할 때마다* 이러한 **설정 작업 시간** 을 줄일 수 있겠죠?

핵심은 *"공통적인 것은 스타터킷에, 특화된 것은 프로젝트마다"* 다. 자주 쓰는 서버는 미리 박아두고, *프로젝트 상황에 맞는* 서버는 그때그때 더하면 — 클로드 코드와의 개발 워크플로우가 **시작부터 한 발 앞서
** 나간다.

## MCP 사용시 주의사항

[앞 챕터](#유용한-mcp-servers-설치)에서 *유용한 서버들을 잔뜩* 심었다. 그런데 — **많이 깔수록 좋은 걸까?** 여기엔 *꼭 알아야 할 함정* 이 하나 있다. 바로 **토큰** 이다.

### MCP 서버는 '공짜'가 아니다 — 컨텍스트를 먹는다

클로드 코드를 켜고 `/context` 로 [컨텍스트](/claude-code-슬래시-명령어와-단축키)를 보면, *지금 쓰고 있는 토큰량* 이 보인다.

> 다행히 현재 사용 중인 MCP가 *적기 때문에* 토큰이 **0.6% 정도만** 사용된 걸 확인할 수 있죠.

문제는 *서버가 많아질 때* 다. MCP 서버는 켜지는 순간 **자기 도구 설명 (툴 정의)** 을 컨텍스트에 밀어 넣는다. 그래서 서버가 많으면 —

> 작업을 *시작하기도 전에* 이미 **상당한 컨텍스트** 를 차지하게 됩니다.

즉 *아무 일도 안 했는데* 이미 토큰이 깎여 있는 상태가 된다. 이게 **MCP 과다 사용의 함정** 이다.

### 양날의 검 — 효과적이지만, 낭비도 크다

> MCP 서버를 활용하면 AI를 활용해서 *보다 더 효과적으로* 개발할 수 있지만, MCP의 *과도한 사용* 은 이러한 **토큰 낭비** 를 초래할 수 있어요.

MCP는 분명 강력하다. 하지만 *켜둔 만큼 비용* 을 치른다. *쓰지도 않는 서버* 가 켜져 있으면, 그건 **순수한 낭비** 다.

### 해결 — 필요한 것만, 그리고 `/clear`·`/compact`

강사가 권하는 방법은 단순하고 명확하다.

| 상황                | 권장 사항                                                                         |
|---------------------|-----------------------------------------------------------------------------------|
| **평소**            | *꼭 필요한 MCP 서버만* 활성화 — 안 쓰는 건 **꺼둔다**                             |
| **많이 써야 할 때** | [`/clear`·`/compact`](/claude-code-슬래시-명령어와-단축키) 명령어를 **적극 활용** |

- **`/clear`** — *대화를 비워* 컨텍스트를 처음 상태로 되돌린다
- **`/compact`** — *대화를 압축* 해 핵심만 남기고 토큰을 회수한다

부득이하게 *여러 서버* 를 동시에 써야 한다면, 작업 사이사이 **컨텍스트를 비우고 압축** 하며 *토큰 여유* 를 확보하는 게 핵심이다.

### 정리하며

MCP 사용 시 주의사항을 정리하면 다음과 같다.

- **함정** → MCP 서버는 켜지는 순간 *툴 정의* 로 **컨텍스트를 미리 잡아먹는다**
- **양날의 검** → *효과적* 이지만, **과도하면 토큰 낭비**
- **원칙** → *꼭 필요한 서버만* 활성화 — 안 쓰는 건 끄기
- **불가피할 땐** → [`/clear`·`/compact`](/claude-code-슬래시-명령어와-단축키)로 **컨텍스트 관리**

[앞 챕터](#유용한-mcp-servers-설치)가 *"좋은 서버를 갖추는"* 이야기였다면, 이번 챕터는 *"갖춘 만큼 절제하라"* 는 이야기다. **많이 깔되, 켜둘 땐 신중하게** — 이 균형이 클로드 코드를
*오래, 효율적으로* 쓰는 비결이다.
